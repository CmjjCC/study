# AI 智能试卷分析与弱点靶向训练系统 — 项目计划书

> 文档版本：v1.0 ｜ 编写日期：2026-07-11
> 项目代号：StudyLoop（学情闭环）

---

## 一、项目概述

### 1.1 项目名称
AI 智能试卷分析与弱点靶向训练核心系统

### 1.2 一句话定位
面向初三至高三学段的全学科一站式升学备考学习网站，以「试卷录入 → 弱点诊断 → 靶向出题 → 刷题训练 → 二次复盘」闭环为核心，实现个性化查漏补缺与精准提分。

### 1.3 项目背景
当前中学生备考普遍存在三大痛点：
- **刷题盲目**：题海战术低效，无法针对个人薄弱点发力；
- **复盘粗放**：试卷讲评笼统，缺乏对"基础漏洞 / 题型短板 / 思路缺陷 / 易错误区"的精细化区分；
- **档案缺失**：学习弱点无沉淀、无追踪，提升节奏难以个性化。

本项目以 AI 为内核，把试卷转化为可量化、可追踪、可训练的学情资产，形成长期个人弱点档案。

### 1.4 项目目标
| 维度 | 目标 |
|------|------|
| 产品 | 上线可用的 Web 站点，覆盖初三~高三全学段全学科（首期语数英物） |
| 闭环 | 打通"录入—诊断—出题—训练—复盘"全链路，弱点档案动态更新 |
| 扩展 | 架构预留化学、生物、历史、地理、政治接口，可低成本接入 |
| 体验 | UI 年轻化、学生风、护眼模式，结构清晰、代码规范、可拓展 |

## 二、用户画像与使用场景

### 2.1 目标用户
- **核心用户**：初三、高一、高二、高三在校学生
- **延伸用户**：学生家长（查看学情报告）、教师（可选，二期）

### 2.2 典型场景
1. **考后复盘**：月考/期中/期末后，上传试卷图片，系统自动诊断薄弱点并生成提升方案。
2. **专项突破**：针对高频失分题型，一键生成同考点变式题进行专项刷题。
3. **长期追踪**：随学习进度动态更新弱点档案，实时查看知识点掌握曲线。
4. **日常学习**：分科学习主页、错题本、单词记忆、素材/模板库等基础能力日常使用。

## 三、功能规划

### 3.1 核心系统（AI 驱动）

#### F1 试卷录入系统
- 支持三种录入方式：
  - 手动上传试卷图片（OCR + AI 识别）
  - 粘贴试卷错题文本
  - 录入考试得分情况（题目—得分—失分原因）
- AI 全自动解析：逐题匹配 **学科 / 年级 / 知识点 / 题型 / 难度**

#### F2 弱点诊断系统
- 问题分类诊断（杜绝笼统评价）：
  - 基础漏洞（知识点未掌握）
  - 题型短板（题型不熟练）
  - 解题思路缺陷（方法/逻辑问题）
  - 应试易错误区（审题、规范、粗心）
- 自动生成 **专属个人弱点分析报告**：
  - 罗列：急需补强知识点 / 高频失分题型 / 知识盲区 / 能力短板
  - 标注：学习优先级
  - 输出：可落地的提升方案（强化知识点、解题方法、答题规范、易错规避要点）

#### F3 靶向出题训练系统
- 基于专属薄弱点，AI 生成 **同考点 / 同难度 / 同题型** 的变式练习题
- 专项突破，避免无效刷题
- 难度自适应：随掌握度动态调整

#### F4 闭环复盘系统
- 训练后自动批改纠错 → 复盘漏洞 → 更新弱点档案
- 闭环：`试卷录入 → 弱点诊断 → 分析总结 → 靶向出题 → 刷题训练 → 二次复盘`
- 永久留存个人学习弱点档案，动态更新学情数据

### 3.2 基础功能模块

| 模块 | 说明 |
|------|------|
| 年级切换系统 | 初三 / 高一 / 高二 / 高三，切换后内容与难度自适应 |
| 分科学习主页 | 各学科独立主页，展示学情概览与入口 |
| 智能刷题系统 | 按"靶向 / 自由 / 错题重练"多模式刷题 |
| 答案详细解析 | 每题给出解题步骤、知识点归属、易错提示 |
| 考点总结 | 按知识点/题型聚合的高频考点卡片 |
| 错题本自动收录 | 失分题自动入库，支持标注、重做、归档 |
| 单词记忆系统 | 艾宾浩斯曲线，背诵+复习+测验 |
| 素材库 | 语文作文素材、英语语料等 |
| 模板库 | 答题模板、作文模板、解题套路 |
| 收藏功能 | 题目、素材、模板收藏管理 |
| 学习记录统计 | 时长、题量、正确率、知识点掌握曲线 |

### 3.3 学科扩展规划
- **一期**：语文、数学、英语、物理（4 学科）
- **二期**：化学、生物（理科补齐）
- **三期**：历史、地理、政治（文科补齐）
- 架构层面通过 **学科适配层（Subject Adapter）** 统一接口，新增学科仅需注册知识图谱与题型模板，不改主干流程。

## 四、技术架构

### 4.1 整体分层
```
┌─────────────────────────────────────────────┐
│              前端（Web 站点）                 │
│   React + TypeScript + TailwindCSS          │
└─────────────────────────────────────────────┘
                      ↕  REST / WebSocket
┌─────────────────────────────────────────────┐
│              后端服务层                       │
│   API 网关 / 鉴权 / 业务编排                  │
├─────────────────────────────────────────────┤
│  AI 引擎层                                   │
│  ├─ 试卷解析（OCR + LLM 结构化）             │
│  ├─ 弱点诊断（规则 + LLM 推理）              │
│  └─ 靶向出题（LLM 生成 + 质量校验）          │
├─────────────────────────────────────────────┤
│  数据层                                       │
│  ├─ 关系库（用户、试卷、题目、错题）          │
│  ├─ 文档库（弱点档案、分析报告）              │
│  └─ 向量库（知识点检索、相似题召回）          │
└─────────────────────────────────────────────┘
```

### 4.2 技术选型（建议）
| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | React 18 + TypeScript + Vite | 生态成熟、组件化、易扩展 |
| UI | TailwindCSS + 自定义组件库 | 年轻化、护眼主题易切换 |
| 后端 | Node.js (NestJS) 或 Python (FastAPI) | AI 集成友好，建议 FastAPI |
| 数据库 | PostgreSQL + MongoDB | 关系数据 + 文档数据分离 |
| 向量库 | Qdrant / pgvector | 知识点与相似题检索 |
| AI | Claude API（试卷解析/诊断/出题）+ 国内 OCR | 多模态理解强，长文本推理稳定 |
| 缓存 | Redis | 会话、热点题、限流 |
| 部署 | Docker + Nginx，可上云 | 可拓展、可移植 |

### 4.3 关键 AI 流程设计

**试卷解析流程**
```
图片 → OCR 文本/公式识别 → LLM 结构化(题号/题干/选项/答案/得分)
     → 知识点标注(向量召回 + LLM 校验) → 入库
```

**弱点诊断流程**
```
错题集 → 按(知识点, 题型, 失分原因)聚类
       → 规则引擎初判 + LLM 深度归因
       → 分类(基础/题型/思路/误区) + 优先级打分
       → 生成报告 + 提升方案
```

**靶向出题流程**
```
薄弱点 → 检索同知识点相似题 → LLM 生成变式题(约束:同考点/难度/题型)
       → 质量校验(知识点对齐/无歧义/答案唯一) → 入题池 → 推送给用户
```

## 五、数据模型（核心实体）

```
User（用户）          —— 学段/年级、偏好、护眼模式
ExamPaper（试卷）     —— 来源、科目、考试时间、得分
Question（题目）      —— 学科、年级、知识点[]、题型、难度、题干、答案、解析
ExamRecord（答题记录）—— 用户×题目：得分、失分原因、错因分类
WeaknessProfile（弱点档案）—— 用户级：知识点掌握度、薄弱分类、优先级、历史曲线
Report（分析报告）    —— 诊断结论、提升方案、生成时间
TrainingSession（训练会话）—— 来源薄弱点、题目列表、批改结果、复盘更新
```

> 设计原则：实体与学科解耦，学科作为属性 + 知识图谱引用，保证可扩展。

## 六、项目阶段划分与里程碑

### 阶段一：地基与骨架（M1，约 3 周）
- 技术选型确认、仓库与 CI/CD 搭建
- 前端脚手架 + UI 基础组件库（含护眼模式）
- 后端框架 + 鉴权 + 年级切换 + 分科主页骨架
- 数据库设计、核心实体建模

### 阶段二：基础学习功能（M2，约 4 周）
- 智能刷题系统、答案解析、考点总结
- 错题本自动收录、收藏功能
- 单词记忆系统、素材库、模板库
- 学习记录统计

### 阶段三：AI 核心闭环（M3，约 5 周）⭐ 核心
- F1 试卷录入（OCR + 解析 + 知识点标注）
- F2 弱点诊断（分类 + 报告 + 提升方案）
- F3 靶向出题（变式题生成 + 质量校验）
- F4 闭环复盘（批改 → 档案更新）

### 阶段四：学科扩展与优化（M4，约 3 周）
- 化学、生物学科接入（验证扩展接口）
- 难度自适应调优、报告体验打磨
- 性能优化、压测

### 阶段五：上线与迭代（M5，约 2 周）
- 内测 → 灰度 → 正式上线
- 用户反馈收集与快速迭代

> 里程碑节点：M1 骨架可跑通 → M2 基础功能可用 → M3 闭环跑通(Alpha) → M4 多学科(Beta) → M5 上线(GA)

## 八、风险评估

| 风险 | 等级 | 应对策略 |
|------|------|----------|
| 试卷 OCR 识别准确率不足（手写/公式） | 高 | 多模态 LLM 直接识图 + OCR 兜底；人工校对入口 |
| AI 诊断笼统、归因不准 | 高 | 规则引擎+LLM 双路；建立诊断质量评测集；用户反馈闭环 |
| 出题质量参差（知识点不对齐/答案错） | 高 | 生成后质量校验关卡；题目入库前抽检；向量召回约束 |
| 知识图谱构建工作量大 | 中 | 首期聚焦语数英物核心知识点；支持渐进式扩充 |
| 学科扩展接口设计不当导致重构 | 中 | 早期抽象 Subject Adapter；先用化学做扩展验证 |
| AI 调用成本与延迟 | 中 | 缓存相似题/报告；分级调用；异步生成 |
| 学生隐私与数据安全 | 中 | 最小化采集、加密存储、合规审查 |

## 九、验收标准

- [ ] 三种试卷录入方式均可用，AI 能正确解析并标注知识点

- [ ] 靶向出题可生成同考点/同难度/同题型变式题，质量校验通过

- [ ] 11 项基础功能全部可用
- [ ] 学科扩展接口经化学学科验证可低成本接入

### 9.2 非功能验收
- [ ] UI 年轻化、学生风、护眼模式可切换
- [ ] 代码结构清晰、规范一致、文档齐全
- [ ] 核心接口响应 P95 < 3s（AI 生成类异步化）
- [ ] 部署容器化，可拓展
